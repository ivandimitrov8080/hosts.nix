* Introduction & Setup Guide

** Overview

This is a literate Emacs configuration written in Org mode, designed for functional web development with a focus on Haskell, Elm, and Nix. The configuration emphasizes:

- Modern completion and discovery interfaces (Vertico, Consult, which-key)
- Powerful navigation and editing tools (Avy, multiple-cursors)
- LSP-based language support via Eglot
- Beautiful visual enhancements (Catppuccin theme, doom-modeline)
- Comprehensive language support for functional programming

** How to Use This Configuration

This configuration uses Org mode's *literate programming* approach. All Emacs Lisp code is contained in =#+begin_src emacs-lisp= blocks within this file.

When you use =emacsWithPackagesFromUsePackage= from Nix, it automatically:
1. Extracts all code blocks from this file (process called "tangling")
2. Combines them into a single =.el= file
3. Uses that as your init file

You're reading the *source* configuration. The actual Emacs Lisp that runs is generated from this file.

** First Time Setup

After installing Emacs with this configuration:

1. *Icons Setup* (GUI only): Run =M-x all-the-icons-install-fonts= once to install icon fonts
2. *Verify LSP Servers*: Ensure nixd, elm-language-server, and haskell-language-server are available
3. *Test Navigation*: Try =C-x C-f= to open files with the new Vertico interface
4. *Explore Keybindings*: Press any prefix key (like =C-c= or =C-x=) and wait for which-key to show options

** Enabling and Disabling Features

To disable a feature without deleting it:

1. Add =:tangle no= to the code block's header:
   #+begin_example
   #+begin_src emacs-lisp :tangle no
   (require 'some-package)
   #+end_src
   #+end_example

2. Rebuild your Emacs configuration with Nix

To re-enable, simply remove =:tangle no= or change it to =:tangle yes=.

* Performance Notes

** Startup Time Optimization

This configuration prioritizes functionality over startup time, but here are ways to improve performance:

*Current Approach:*
- All packages are loaded eagerly at startup
- LSP (Eglot) is enabled automatically for supported modes
- Tree-sitter is enabled globally

*Optimization Strategies:*

1. *Lazy Loading*: Convert =require= statements to =autoload= or use =use-package= with =:defer=
2. *Conditional LSP*: Remove =eglot-ensure= from mode hooks and call manually when needed (=M-x eglot=)
3. *Disable Heavy Features*: Tree-sitter and doom-modeline can add startup time
4. *Reduce Package Count*: Disable packages you don't use by adding =:tangle no= to their sections

** Profiling Your Configuration

To measure startup time:

#+begin_example
emacs --debug-init
#+end_example

To profile which packages take longest:

#+begin_example
M-x emacs-init-time
#+end_example

For detailed profiling, consider using the =esup= package (not included by default).

** Summary of Custom Keybindings

This configuration adds many custom keybindings. Here are the most important ones:

*** Navigation & Search
- =C-s= - consult-line (search in buffer with live preview)
- =C-x b= - consult-buffer (switch buffers with enhanced interface)
- =C-x C-r= - consult-recent-file (open recent files)
- =M-g i= - consult-imenu (navigate by functions/headings)
- =M-g g= - consult-goto-line (jump to line number)
- =M-s g= - consult-grep (grep in project)
- =M-s r= - consult-ripgrep (faster grep with ripgrep)

*** Character/Word Navigation (Avy)
- =C-:= - avy-goto-char (jump to any visible character)
- =C-'= - avy-goto-char-2 (jump using two characters)
- =M-g f= - avy-goto-line (jump to any visible line)
- =M-g w= - avy-goto-word-1 (jump to word)

*** Editing
- =C->= - mc/mark-next-like-this (add cursor to next match)
- =C-<= - mc/mark-previous-like-this (add cursor to previous match)
